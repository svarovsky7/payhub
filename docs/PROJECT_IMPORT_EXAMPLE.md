# Примеры файлов для импорта проектов

## Структура импорта

**Важно**: Каждое альтернативное название проекта должно быть в отдельной строке с кодом проекта.

| Код | Связанные названия |
|-----|-------------------|
| P-1 | Alt-1 |
| P-1 | Alt-2 |
| P-2 | Alt-3 |

На один код проекта может быть **несколько строк** с разными названиями.

## Примеры

### Пример 1: Жилые комплексы

| Код | Связанные названия |
|-----|-------------------|
| ZhK-001 | Жилой комплекс "Сонет" |
| ZhK-001 | Комплекс Сонет |
| ZhK-001 | Sonnet Complex |
| ZhK-001 | ЖК Сонет |
| ZhK-002 | Торговый центр "Солнечный" |
| ZhK-002 | ТЦ Солнечный |
| ZhK-002 | Sunny Mall |

**Результат**: 2 проекта, первый с 4 названиями, второй с 3.

### Пример 2: Промышленные объекты

| Код | Связанные названия |
|-----|-------------------|
| PROD-05 | Завод №5 |
| PROD-05 | Factory 5 |
| PROD-05 | Fabrica 5 |
| PROD-10 | Завод №10 |
| PROD-10 | Factory 10 |

**Результат**: 2 проекта с названиями.

### Пример 3: Офисные парки

| Код | Связанные названия |
|-----|-------------------|
| TECH-PARK | Технопарк |
| TECH-PARK | Tech Park |
| TECH-PARK | ОП Технопарк |
| OFFICE-A | Офисное здание А |
| OFFICE-A | Building A |
| OFFICE-A | ОЗ-А |

## Процесс импорта

1. Система читает файл Excel
2. Группирует строки по коду проекта
3. Для каждого кода:
   - Ищет существующий проект в БД
   - Если найден - обновляет альтернативные названия
   - Если нет - создает новый проект с кодом как название
4. Заменяет все старые названия на новые из файла

## Правила

✅ **Правильно:**
```
P-1 | Alt Name 1
P-1 | Alt Name 2
P-1 | Alt Name 3
P-2 | Another Name
```

❌ **Неправильно:**
```
P-1 | Alt Name 1; Alt Name 2; Alt Name 3
```

## Важные замечания

- **Обязательные поля**: Код и Связанные названия
- **Дублирование**: Одинаковые названия для одного кода импортируются один раз
- **Существующие проекты**: Если проект с таким кодом уже есть, названия обновляются
- **Пробелы**: Автоматически удаляются с начала и конца

## Структура при экспорте

Когда экспортируете, каждое альтернативное название будет на отдельной строке:

| Код | Название | Связанные названия | Описание | Статус |
|-----|----------|-------------------|---------|--------|
| P-001 | Проект 1 | Alt1 | Описание | Активен |
| P-001 | Проект 1 | Alt2 | Описание | Активен |
| P-002 | Проект 2 | — | Описание | Неактивен |

## Советы

- Используйте Excel для создания файлов
- Сохраняйте в формате `.xlsx` (не `.xls`)
- Проверьте что коды уникальны для каждого проекта
- Названия могут быть на разных языках
- До 1000+ строк в файле
